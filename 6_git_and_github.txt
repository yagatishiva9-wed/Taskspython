----------------------------------------------------------------
WHAT IS GIT?
----------------------------------------------------------------
Git is a free and open-source distributed version control system
(DVCS) created by Linus Torvalds in 2005. It was originally
designed to manage the source code of the Linux kernel.

Git tracks changes to files over time, allowing multiple
developers to collaborate on a project without overwriting each
other's work. Every developer has a full copy of the repository
(including its history) on their local machine.

Key characteristics of Git:
  - Distributed: No single point of failure; every clone is
    a full backup of the project.
  - Fast: Most operations are local, requiring no network.
  - Integrity: Every change is checksummed with SHA-1 hashes.
  - Non-linear: Supports thousands of parallel branches.


----------------------------------------------------------------
WHAT IS GITHUB?
----------------------------------------------------------------
GitHub is a cloud-based hosting platform for Git repositories.
Founded in 2008 and acquired by Microsoft in 2018, GitHub adds
a powerful web interface, collaboration tools, and social coding
features on top of Git.

Think of it this way:
  Git  = the engine (version control tool)
  GitHub = the platform (hosting + collaboration layer)

Other similar platforms: GitLab, Bitbucket, Azure DevOps.


----------------------------------------------------------------
CORE GIT CONCEPTS
----------------------------------------------------------------

Repository (Repo)
  A folder tracked by Git. Contains all your project files and
  the full history of every change ever made.

Commit
  A snapshot of your changes at a point in time. Each commit
  has a unique SHA hash, an author, a timestamp, and a message.

Branch
  A lightweight pointer to a commit. Branches let you work on
  features or fixes in isolation without affecting the main code.
    - main / master : the default, production-ready branch
    - feature branch : created for a specific new feature
    - hotfix branch  : created to fix a critical bug quickly

Merge
  Combining changes from one branch into another.

Pull Request (PR)
  A GitHub feature to propose your branch's changes and request
  a code review before merging into the main branch.

Clone
  Download a full copy of a remote repository to your machine.

Fork
  Create a personal copy of someone else's repository on GitHub
  so you can experiment without affecting the original.

Staging Area (Index)
  A preparation zone where you collect changes before committing.

HEAD
  A pointer to the current commit you are working on.

Remote
  A version of your repository hosted on a server (e.g., GitHub).
  Common remote names: origin, upstream.


----------------------------------------------------------------
ESSENTIAL GIT COMMANDS
----------------------------------------------------------------

SETUP
  git config --global user.name  "Your Name"
  git config --global user.email "you@example.com"

START A PROJECT
  git init                  Initialize a new local repository
  git clone <url>           Clone an existing remote repository

DAILY WORKFLOW
  git status                Show changed / untracked files
  git add <file>            Stage a specific file
  git add .                 Stage all changes
  git commit -m "message"   Commit staged changes with a message
  git push origin <branch>  Push commits to GitHub
  git pull origin <branch>  Fetch + merge remote changes

BRANCHING
  git branch                List all local branches
  git branch <name>         Create a new branch
  git checkout <name>       Switch to a branch
  git checkout -b <name>    Create and switch in one step
  git merge <branch>        Merge a branch into the current one
  git branch -d <name>      Delete a branch

HISTORY & INSPECTION
  git log                   Full commit history
  git log --oneline         Compact one-line history
  git diff                  Show unstaged changes
  git diff --staged         Show staged changes vs last commit
  git show <hash>           Show details of a specific commit

UNDOING CHANGES
  git restore <file>        Discard working-directory changes
  git reset HEAD <file>     Unstage a file
  git revert <hash>         Create a new commit that undoes one
  git stash                 Temporarily shelve uncommitted changes
  git stash pop             Re-apply the most recent stash


----------------------------------------------------------------
GITHUB-SPECIFIC FEATURES
----------------------------------------------------------------

Issues
  Track bugs, enhancements, and tasks. Can be assigned, labeled,
  and linked to pull requests.

Pull Requests (PRs)
  The heart of GitHub collaboration. Propose changes, discuss
  them line by line, request reviewers, and merge when approved.

GitHub Actions
  A built-in CI/CD (Continuous Integration / Continuous
  Deployment) platform. Automate testing, building, and
  deployment with YAML workflow files.

GitHub Pages
  Host static websites (HTML/CSS/JS) directly from a repository
  for free. Ideal for portfolios and documentation.

GitHub Codespaces
  A cloud-based development environment. Code in VS Code right
  in your browser without any local setup.

Wikis & Discussions
  Built-in documentation and community forum features per repo.

Security Features
  Dependabot alerts, secret scanning, and code scanning to
  keep your project safe.


----------------------------------------------------------------
GIT WORKFLOW MODELS
----------------------------------------------------------------

1. Centralized Workflow
   Everyone commits to a single main branch. Simple but
   prone to conflicts on large teams.

2. Feature Branch Workflow
   Each new feature lives on its own branch. PRs are used
   to merge back into main. Most common model.

3. Gitflow Workflow
   Uses dedicated branches: main, develop, feature/*, release/*,
   hotfix/*. Great for projects with scheduled releases.

4. Forking Workflow
   Contributors fork the repo, make changes, and open PRs.
   Standard model for open-source projects.
